# JavaGo

JavaGo is a JVM-compatible transpiler demo that accepts `.javag` code, converts it into Java, and can optionally compile and run that generated code in a short-lived backend workspace.

This backend is designed for a controlled portfolio/demo deployment on Render. It is intentionally minimal and explicitly not a production sandbox.

## Backend Scope

The Render backend can:

- accept JavaGo code
- transpile `.javag` to `.java`
- compile generated Java with `javac`
- run generated bytecode with a strict 5-second timeout
- return transpiled code, compiler output, runtime output, and timeout messages
- delete generated files after each request

It does **not** provide real sandbox isolation.

## Project Layout

```text
src/main/java/javago/
├── JavaGoApplication.java
├── controller/
├── service/
├── transpiler/
├── execution/
├── security/
└── util/
```

Main backend pieces:

- `controller/JavaGoController.java`: REST endpoints
- `service/JavaGoService.java`: request orchestration
- `transpiler/JavaGoTranspiler.java`: JavaGo to Java conversion
- `execution/ExecutionService.java`: `javac` and `java` execution via `ProcessBuilder`
- `security/CodeSafetyFilter.java`: simple V1 blocked-keyword scan
- `util/WorkspaceManager.java`: temporary workspace creation and cleanup

Legacy desktop tooling from the earlier project phase is still present elsewhere in the repository, but the deployable backend path is the Spring Boot application under `src/main/java`.

## API Endpoints

### `POST /transpile`

Request:

```json
{
  "code": "class Demo { main() { Go.println(\"Hello\"); } }"
}
```

Response:

```json
{
  "success": true,
  "transpiledCode": "public class ...",
  "compilerErrors": null,
  "runtimeOutput": null,
  "runtimeErrors": null,
  "timedOut": false,
  "warnings": [],
  "message": "Transpile completed."
}
```

### `POST /compile`

Request:

```json
{
  "code": "class Demo { main() { Go.println(\"Hello\"); } }"
}
```

### `POST /run`

Request:

```json
{
  "code": "Go.utilities(Scanner input); class Demo { main() { String name = input(); Go.println(name); } }",
  "input": "Parneet\n"
}
```

## Safety Model

Before compile/run, the backend rejects code containing these blocked fragments:

- `Runtime`
- `ProcessBuilder`
- `File`
- `Socket`
- `System.exit`

This is only a simple string filter for a demo deployment. It is not a real security boundary.

## Execution Model

For each compile or run request the backend:

1. creates a temporary workspace using `Files.createTempDirectory(...)`
2. writes generated Java into that workspace
3. compiles with `javac`
4. runs with `java` if compilation succeeds
5. kills the process if execution exceeds 5 seconds
6. deletes the temporary directory and generated files

On Linux hosts such as Render, those temporary folders are typically created under `/tmp`.

## Local Run

Build:

```bash
mvn clean package
```

Run:

```bash
java -jar target/javago-0.0.1-SNAPSHOT.jar
```

Default port:

```text
8080
```

Render will inject `PORT`, and the backend reads it through:

```properties
server.port=${PORT:8080}
```

## Render Notes

The repository includes a minimal `render.yaml`:

```yaml
services:
  - type: web
    name: javago-backend
    env: java
    plan: free
    buildCommand: mvn clean package -DskipTests
    startCommand: java -jar target/javago-0.0.1-SNAPSHOT.jar
```

For deployment, make sure the Render service uses a JDK runtime so `javac` is available in addition to `java`.

## Important Limitation

This backend is suitable for a portfolio demo, classroom demo, or controlled showcase. It should not be represented as a secure online compiler platform.
